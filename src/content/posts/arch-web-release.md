# 🖥️ Arch-web 勉强能用了

> **项目地址：** [https://cxl--package.top/](https://cxl--package.top/)
>
> **状态：** 能跑，但别问怎么跑的

---

## 一、这是什么？

**Arch-web** 是一个运行在浏览器里的 **Arch Linux 模拟器**。

对，你没看错。不是远程桌面，不是 VNC，是**纯前端**——用 React + Vite 手搓出来的一个"看起来像 Arch"的网页。

它有一个桌面、一个终端、一些"软件"、还有一些动态壁纸。虽然里面的 `neofetch` 不会真的读取你的 CPU 信息，里面的 `pacman -Syu` 不会真的更新你的系统，但**它看起来很像那么回事**。

这就够了。

> "在程序员的世界里，如果一个东西看起来像鸭子，叫起来像鸭子，那它就是鸭子——即使它其实是一只 React 组件。"

---

## 二、它有多慢？

**首次打开：1-3 分钟。**

不是加载慢，是**你的浏览器在怀疑人生**。它要消化 600+ 个 npm 依赖、解析一个 React 屎山、加载几张动态壁纸，然后还要问自己：

> "我真的要在浏览器里运行一个 Linux 桌面吗？"

答案是：是的，浏览器，你跑也得跑，不跑也得跑。

**动态壁纸加载：** 可能更久。

因为壁纸是从外部来源加载的，如果你的网络正在和 Cloudflare 进行一场**谁先放弃的博弈**，那壁纸可能会迟到，但永远不会缺席——大概。

> **网站都免费了，还要啥自行车？慢慢加载得了。**

## 三、哪些软件能用？

这是一个**薛定谔的 Linux**。有些软件能用，有些不能，取决于：

1. 我当时有没有写那个功能
2. 我当时有没有写崩那个功能
3. 我当时有没有因为写崩而删了那个功能

### ✅ 能用的（大概）

| 软件 | 状态 | 说明 |
|------|------|------|
| **终端** | ✅ 能用 | 能输入命令，能显示输出，但别真的以为它在执行 |
| **neofetch** | ✅ 能装 | 显示的是预设信息，不是你的真实硬件 |
| **文件管理器** | ✅ 能点 | 可以新建文件夹，但刷新就没了（因为没后端） |
| **动态壁纸** | ⚠️ 能加载 | 看网络心情，有时候是壁纸，有时候是空白 |
| **设置面板** | ✅ 能打开 | 能切换主题，但切换完可能要等一会儿 |

### ❌ 不能用的（别试）

| 软件 | 状态 | 说明 |
|------|------|------|
| **sudo rm -rf /** | ❌ 不能 | 浏览器不会让你删系统，但会卡死 |
| **pacman -Syu** | ❌ 不能 | 不会真的更新，只是打印一行字 |
| **vim** | ❌ 没有 | 我懒得实现，用 nano 吧（也没有） |
| **Chrome** | ❌ 不能 | 在浏览器里运行浏览器？你想让我死？ |
| **Steam** | ❌ 不能 | 但你可以打开 Lutris 的图标假装自己在玩游戏 |
| **Hyprland** | ❌ 不能 | 这是网页，不是 Wayland 合成器，醒醒 |

## 四、技术栈（ aka 屎山成分表）

```
React 18          —— 因为 Vue 我不会
Vite              —— 因为 Webpack 太慢了
Electron          —— 因为"也许有一天"我会打包桌面版
Tailwind CSS      —— 因为写 CSS 太烦了
600+ npm 依赖     —— 因为"这个库看起来有用"
动态壁纸 (swww)   —— 因为桌面不能是黑的
我的耐心          —— 已耗尽
```

构建时间：**取决于你的电脑和 Cloudflare 的心情。**

在本地 `npm run dev` 的时候，Vite 热更新有时候比我的反射弧还慢。但 `npm run build` 的时候，Cloudflare 的构建缓存会救我一命——如果它没睡着的话。

## 五、使用指南

### 首次访问
1. 打开 [https://cxl--package.top/](https://cxl--package.top/)
2. 等待 1-3 分钟（可以去泡杯茶）
3. 如果看到桌面，恭喜你，成功了
4. 如果看到白屏，请刷新，再等待 1-3 分钟
5. 如果还是白屏，说明 Cloudflare 在睡觉，请明天再来

### 基本操作
- **点击图标**：打开"软件"（有些是空的，别问）
- **右键桌面**：可能有菜单，也可能没有
- **终端输入命令**：会显示预设输出，不会真的执行
- **切换壁纸**：如果加载出来了，可以换；没加载出来就盯着黑屏冥想

### 注意事项
- 不要尝试 `sudo` 任何命令，浏览器没有 root 权限（大概）
- 不要同时打开太多窗口，你的内存会哭
- 不要在手机上用，除非你想体验"拇指戳屏幕戳到抽筋"
- 不要期待它能代替真正的 Arch，它只是一个**赛博手办**
